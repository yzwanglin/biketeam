# 手机无法访问局域网服务器 - 排查指南

## 🎯 问题现象
- ✅ 电脑访问 `http://192.168.11.156:8000/user-app.html` 正常
- ❌ 手机访问同一地址失败

---

## 🔍 排查步骤（按顺序执行）

### 步骤1：确认手机和电脑在同一WiFi网络

**在手机上检查：**
- 设置 → WiFi → 确认连接的WiFi名称
- 必须与电脑连接的WiFi完全一致

**常见错误：**
- 电脑连网线，手机连WiFi（可能不是同一个网络）
- 手机使用移动数据（4G/5G）
- 连接了不同的路由器（如访客网络）

---

### 步骤2：检查Mac防火墙设置

**在Mac上执行：**

```bash
# 查看防火墙状态
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --getglobalstate

# 如果显示"Firewall is enabled"，需要添加例外或临时关闭
# 临时关闭防火墙（测试用）
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --setglobalstate off

# 测试完成后重新开启
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --setglobalstate on
```

**或者通过图形界面：**
1. 系统偏好设置 → 安全性与隐私 → 防火墙
2. 点击"关闭防火墙"（临时测试）
3. 或点击"防火墙选项" → 添加Python/http.server

---

### 步骤3：检查服务器绑定地址

**问题：** 服务器可能只绑定了localhost，没有监听所有网卡

**解决方案：**

确保使用 `--bind 0.0.0.0` 参数启动服务器：

```bash
cd /Users/apple/WorkBuddy/20260310225816

# 正确的方式（监听所有网卡）
python3 -m http.server 8000 --bind 0.0.0.0

# 或使用Node.js
npx http-server -p 8000 -a 0.0.0.0
```

**验证服务器是否监听正确：**
```bash
# 查看8000端口的监听情况
lsof -i :8000

# 应该显示监听 0.0.0.0:8000 或 *:8000
# 如果只显示 127.0.0.1:8000，则手机无法访问
```

---

### 步骤4：检查IP地址是否正确

**获取准确的局域网IP：**

```bash
# 方法1
ipconfig getifaddr en0

# 方法2
ifconfig | grep "inet " | grep -v 127.0.0.1

# 方法3（查看所有网络接口）
ifconfig
```

**注意：**
- 如果电脑同时连接了网线和WiFi，可能有多个IP
- 确保使用WiFi对应的IP地址（通常是en0）
- 192.168.11.156 是否正确？请再次确认

---

### 步骤5：测试网络连通性

**在手机上测试（使用浏览器或网络工具APP）：**

1. **Ping测试：**
   - 下载"网络百宝箱"或"Ping工具"APP
   - Ping 192.168.11.156
   - 看是否能通

2. **端口测试：**
   - 使用"端口扫描"类APP测试 192.168.11.156:8000

**在电脑上测试手机是否可达：**
```bash
# 查看手机IP（需要在手机上查看WiFi详情）
# 假设手机IP是 192.168.11.xxx
ping 192.168.11.xxx
```

---

### 步骤6：检查路由器AP隔离设置

**问题：** 某些路由器开启了"AP隔离"或"客户端隔离"

**解决方案：**
1. 登录路由器管理界面（通常是192.168.11.1）
2. 查找"AP隔离"、"客户端隔离"、"无线隔离"等选项
3. 关闭该功能

**常见路由器设置位置：**
- TP-Link：无线设置 → 无线高级设置 → AP隔离
- 小米：常用设置 → 局域网设置 → AP隔离
- 华为：更多功能 → WiFi设置 → AP隔离

---

### 步骤7：使用替代方案

如果以上方法都无效，尝试：

#### 方案A：使用ngrok内网穿透
```bash
# 安装ngrok
brew install ngrok

# 启动隧道
ngrok http 8000

# 会生成一个公网URL，如 https://xxxx.ngrok.io
# 手机和电脑都可以访问这个地址
```

#### 方案B：使用Cloudflare Tunnel
```bash
# 安装cloudflared
brew install cloudflared

# 创建隧道
cloudflared tunnel --url http://localhost:8000
```

## 💡 常见解决方案总结

**90%的问题通过以下3步解决：**

1. **确保同一WiFi**（最常见错误）
2. **关闭Mac防火墙**（系统偏好设置 → 安全性）
3. **使用正确IP**（用 `ipconfig getifaddr en0` 确认）

**如果还是不行，请告诉我：**
- 手机和电脑是否确实连接了同一个WiFi？
- Mac防火墙是否已关闭？
- 服务器启动命令是什么？
