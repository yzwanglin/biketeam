# 🚲 常骑骑GPS设备技术对接文档

## 一、概述

本文档描述博实结KG12 GPS定位器与"常骑骑"骑行积分平台的技术对接方案。KG12支持JT/T 808部标协议，可实现车辆实时定位、轨迹追踪、里程计算等功能。

## 三、JT/T 808协议详解

### 3.1 协议概述

JT/T 808是中国交通运输部发布的《道路运输车辆卫星定位系统终端通讯协议及数据格式》行业标准。

### 3.2 数据包结构

```
[标识位 0x7E] [消息头] [消息体] [校验码] [标识位 0x7E]
```

### 3.3 消息头结构

| 字段 | 长度 | 说明 |
|------|------|------|
| 消息ID | 2字节 | 指令类型 |
| 消息属性 | 2字节 | 数据长度、分包标识 |
| 终端手机号 | 6字节 | BCD编码 |
| 消息流水号 | 2字节 | 顺序递增 |

### 3.4 主要消息ID

| 消息ID | 名称 | 方向 |
|--------|------|------|
| 0x0001 | 终端通用应答 | 终端→平台 |
| 0x0002 | 平台通用应答 | 平台→终端 |
| 0x0100 | 终端注册 | 终端→平台 |
| 0x0102 | 终端鉴权 | 终端→平台 |
| 0x0200 | 位置信息汇报 | 终端→平台 |
| 0x0201 | 位置信息查询 | 平台→终端 |
| 0x0302 | 文本信息下发 | 平台→终端 |
| 0x0303 | 事件设置 | 平台→终端 |

## 四、数据对接方案

### 4.1 系统架构

```
┌─────────────────┐      TCP/UDP       ┌─────────────────┐
│  博实结KG12     │ ←──────────────→  │  常骑骑服务器   │
│  GPS定位设备    │   JT808协议       │  (数据接收端)   │
└─────────────────┘                    └─────────────────┘
                                                │
                                                ▼
                                        ┌─────────────────┐
                                        │   数据库存储    │
                                        └─────────────────┘
                                                │
                                                ▼
                                        ┌─────────────────┐
                                        │   APP前端展示   │
                                        └─────────────────┘
```

### 4.2 数据流设计

1. **设备 → 服务器**
   - GPS设备通过4G网络发送位置数据
   - 使用TCP/UDP传输JT808协议数据包
   - 服务器解析数据包，提取位置信息

2. **数据处理**
   - 解析经纬度、速度、方向、时间
   - 计算骑行里程
   - 判断骑行状态（开始/进行中/结束）

3. **存储与应用**
   - 存储轨迹数据到数据库
   - 计算积分并更新用户信息
   - APP实时展示骑行数据

## 五、关键数据字段

### 5.1 位置信息汇报（0x0200）

| 字段 | 字节数 | 说明 |
|------|--------|------|
| 报警标志 | 4字节 | bit位定义 |
| 状态 | 4字节 | bit位定义 |
| 纬度 | 4字节 | 以度为单位的整数 |
| 经度 | 4字节 | 以度为单位的整数 |
| 高度 | 2字节 | 海拔高度(米) |
| 速度 | 2字节 | 1/10km/h |
| 方向 | 2字节 | 0-359度 |
| 时间 | 6字节 | BCD码，YYMMDDHHMMSS |

### 5.2 数据解析示例

```javascript
// 解析位置信息
function parseLocationMessage(msgBody) {
    // 报警标志 (4字节)
    const alarm = msgBody.readUInt32LE(0);
    
    // 状态 (4字节)
    const status = msgBody.readUInt32LE(4);
    
    // 纬度 (4字节) - 需要除以10^6得到度
    const latitude = msgBody.readUInt32LE(8) / 1000000;
    
    // 经度 (4字节) - 需要除以10^6得到度
    const longitude = msgBody.readUInt32LE(12) / 1000000;
    
    // 高度 (2字节)
    const altitude = msgBody.readUInt16LE(16);
    
    // 速度 (2字节) - 需要除以10得到km/h
    const speed = msgBody.readUInt16LE(18) / 10;
    
    // 方向 (2字节)
    const direction = msgBody.readUInt16LE(20);
    
    // 时间 (6字节) BCD码
    const time = parseBcdTime(msgBody.slice(22, 28));
    
    return {
        alarm,
        status,
        latitude,
        longitude,
        altitude,
        speed,
        direction,
        time
    };
}
```

## 六、骑行状态判断逻辑

### 6.1 骑行开始判断

| 条件 | 说明 |
|------|------|
| 速度 > 0 | 检测到移动 |
| 速度阈值 | > 5 km/h（排除抖动） |
| 持续时间 | 速度>0持续30秒以上 |
| 位置变化 | 距离变化 > 50米 |

### 6.2 骑行结束判断

| 条件 | 说明 |
|------|------|
| 速度 = 0 | 停止移动 |
| 持续时间 | 速度=0持续60秒以上 |
| 状态确认 | 确认用户点击"结束骑行" |

### 6.3 里程计算

```javascript
// 使用Haversine公式计算两点距离
function calculateDistance(lat1, lon1, lat2, lon2) {
    const R = 6371; // 地球半径(km)
    const dLat = toRadians(lat2 - lat1);
    const dLon = toRadians(lon2 - lon1);
    const a = Math.sin(dLat/2) * Math.sin(dLat/2) +
              Math.cos(toRadians(lat1)) * Math.cos(toRadians(lat2)) *
              Math.sin(dLon/2) * Math.sin(dLon/2);
    const c = 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1-a));
    return R * c; // 距离(km)
}
```

## 八、服务器端实现

### 8.1 技术栈

- **语言**: Node.js / Python
- **框架**: Express / Flask
- **数据库**: MySQL / PostgreSQL
- **协议**: TCP Server (net模块)

### 8.2 核心代码示例（Node.js）

```javascript
const net = require('net');

const server = net.createServer((socket) => {
    console.log('设备连接:', socket.remoteAddress);
    
    socket.on('data', (data) => {
        // 解析JT808数据包
        const messages = parseJ808Messages(data);
        
        messages.forEach(msg => {
            switch (msg.id) {
                case 0x0100: // 终端注册
                    handleRegister(socket, msg);
                    break;
                case 0x0102: // 终端鉴权
                    handleAuth(socket, msg);
                    break;
                case 0x0200: // 位置信息汇报
                    handleLocation(msg);
                    break;
            }
        });
    });
    
    socket.on('error', (err) => {
        console.error('连接错误:', err);
    });
});

server.listen(8080, '0.0.0.0', () => {
    console.log('JT808服务器启动，监听端口8080');
});
```

## 九、注意事项

### 9.1 设备配置

1. **服务器地址配置**
   - 需要将KG12设备配置为连接常骑骑服务器
   - 需要设备商提供配置工具或远程配置

2. **参数设置**
   - 心跳间隔：30秒
   - 位置上报间隔：10-30秒
   - 定位模式：北斗+GPS

### 9.2 SIM卡要求

| 项目 | 要求 |
|------|------|
| 网络制式 | 4G LTE |
| 流量套餐 | 建议100MB/月以上 |
| 运营商 | 移动/联通/电信均可 |

### 9.3 安全考虑

1. **数据加密**
   - 设备与服务器通信应使用加密传输
   - 敏感数据应加密存储

2. **身份鉴权**
   - 设备注册时进行身份验证
   - 定期更新鉴权码
